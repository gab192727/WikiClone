About

Last updated: November 1, 2025

Wikiclone is a Django-powered Wikipedia clone built as an educational project based on Harvard’s CS50 Web course. It’s designed to help me (Gabriel Eric Tope) practice and demonstrate Django, parsing, caching, and API handling.

Project Overview

  • Project name: Wikiclone
  • One-line summary: A Django-powered Wikipedia clone.
  • Inspiration: Built on top of CS50’s Wiki project (CS50 Web Programming with Python & JavaScript).

Purpose

  • Why: To strengthen Django skills (caching, API handling, parsing), and to learn production patterns.
  • Who it’s for: Learners, developers, and anyone curious about how a wiki-like site is built.
  • Distribution: Public on GitHub with walkthrough videos planned on YouTube.

How It Works

The app parses simple wiki-style markup (headers using =), converts sections to styled HTML, auto-links internal references, handles disambiguation pages, and renders articles with Bootstrap containers.

  • Header & section parsing (regex-based)
  • Auto-linking internal references
  • Disambiguation handling and special intro layout
  • Custom section rendering with graceful fallbacks
  • Random article feature
  • Server-side caching and API utilities (see utils.py)
  • Dark mode support in the UI (CSS toggle-ready)

Technologies used: Django, Python, regex, HTML/CSS, JavaScript, Bootstrap

Development Journey

  • Key learnings: API caching strategies, robust parsing logic in utils.py, and safe HTML rendering in templates.
  • Main challenge: Parsing and reliable API integration — about half the development time was spent refining this.
  • Planned improvements: Enhanced search & category browsing, UI polish, category tagging, and more resilient API error handling.
